# Findash — Financial Dashboard Design

**Date:** 2026-04-01

## Overview

A personal financial dashboard web app that visualises personal finance and investment transactions. The user (and anyone who forks/deploys it) maintains a lightweight JSON file of transactions, updated locally via Claude Code. The app handles all processing and visualisation client-side. Raw financial data never enters the public repo.

---

## Data

### `transactions.json` (Vercel Blob, private)

```json
{
  "transactions": [
    {
      "id": "uuid",
      "date": "2025-03-15",
      "amount": -45.50,
      "type": "expense",
      "category": "Food & Drink",
      "description": "Whole Foods",
      "account": "Checking"
    },
    {
      "id": "uuid",
      "date": "2025-03-10",
      "amount": -500.00,
      "type": "investment",
      "category": "Investment",
      "description": "Buy AAPL",
      "account": "Brokerage",
      "ticker": "AAPL",
      "shares": 2.5,
      "price_per_share": 200.00
    }
  ]
}
```

- `type`: `income | expense | transfer | investment`
- `amount`: negative = money out, positive = money in
- `ticker`, `shares`, `price_per_share`: investment transactions only

### `insights.json` (Vercel Blob, private)

Generated by Claude Code alongside `transactions.json` updates.

```json
{
  "generated_at": "2025-03-31T12:00:00Z",
  "monthly": [
    {
      "month": "2025-03",
      "narrative": "You spent 23% more on dining in March...",
      "savings_rate": 0.18,
      "highlights": ["Highest spending month in Q1", "Salary received on time"]
    }
  ],
  "anomalies": [
    {
      "date": "2025-03-03",
      "description": "Unusual $800 charge at Best Buy",
      "severity": "medium"
    }
  ],
  "trends": [
    {
      "type": "savings_rate",
      "description": "Savings rate improved 3 months in a row",
      "direction": "positive"
    }
  ],
  "investments": [
    {
      "ticker": "AAPL",
      "commentary": "Position up 12%, portfolio concentration risk in tech sector"
    }
  ]
}
```

**Update workflow:** User provides statements/screenshots to Claude Code locally → Claude updates both JSON files → User pushes to Vercel Blob via CLI or dashboard. Raw statements never leave the user's machine.

## Architecture

```
Public Repo (code only — no financial data)
└── Next.js App Router (Vercel)
    ├── middleware.ts          ← password gate on all routes
    ├── /login                 ← password form
    ├── /api/transactions      ← server route, reads from Vercel Blob
    ├── /api/insights          ← server route, reads from Vercel Blob
    └── /dashboard/*           ← all pages (client components)
        ├── /                  ← Overview
        ├── /transactions      ← Transaction table/list
        ├── /spending          ← Category breakdowns
        ├── /investments       ← Portfolio & holdings
        └── /insights          ← AI summaries & anomalies

Vercel (private)
├── Blob: transactions.json
├── Blob: insights.json
├── Env: PASSWORD_HASH         ← bcrypt hash of user's chosen password
└── Env: BLOB_READ_WRITE_TOKEN
```

All data fetching is server-side. Only processed/aggregated data reaches the client.

---

## Auth

- Next.js middleware checks all routes for a valid signed session cookie
- No session → redirect to `/login`
- Login: single password field, checked against `PASSWORD_HASH` env var (bcrypt)
- Success: sets secure, httpOnly, sameSite=strict cookie, 30-day expiry
- Anyone forking the repo sets their own `PASSWORD_HASH` + Blob token in Vercel env vars

## Dashboard Pages

| Page | Key Content |
|------|-------------|
| **Overview** | Net worth card, monthly cash flow bar chart, spending donut, quick stats (income, expenses, savings rate) |
| **Transactions** | Searchable/filterable table (desktop) / card list (mobile) — filter by date, category, type, account |
| **Spending** | Category breakdown over time, monthly trend lines, top merchants |
| **Investments** | Holdings table with P&L, portfolio allocation donut, value over time line chart |
| **Insights** | Monthly narrative cards, anomaly alerts with severity badges, trend indicators, investment commentary |

## UI/UX

- **Design system:** shadcn/ui + Tailwind CSS
- **Charts:** Recharts
- **Date handling:** date-fns
- **Style:** Dark mode by default, clean financial aesthetic via ui-ux-pro-max skill
- **Responsive:**
  - Desktop: sidebar nav + main content
  - Mobile: bottom tab bar, stacked cards, chart simplification, card list for transactions
- All aggregation/processing done client-side from fetched JSON
